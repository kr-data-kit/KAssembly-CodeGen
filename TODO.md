- `task`를 없애고, cli 개선하기

- 패키지 이름을 안정하게 체크하는 것이 필요함

- `go.mod` 생성하기

- `fmt` 를 `log`으로 변환하기

- service 구조체가 값이 빠져도 gen이 가능하게

- 패키지 이름 바꾸기 (codegen)

- go.mod 추가 여부 설정 (+ 레포 생성 여부)

- README 다듬기

# 고민

- `build` 와 `generate`를 이미 cli로 제공하고 있는데 굳이 설정을 `task`로 할 이유가 있는지 고민하기
: 일반 사용자에겐 만들어진 결과물만을 제공해주고, 개발할려는 사용자에겐 cli를 사용 하게끔 지원하는게 더 나을 것 같다.

- 향후에 `generate`를 통해 만든 코드를 다른 하나의 레포로 지원해야 할듯 : 그럴려면 `go.mod`를 생성하는 걸 만들어야 함
